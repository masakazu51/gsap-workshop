GSAPの基本的な使い方を学びましょう。 GSAPは直感的なAPIで、少ないコードで豊かなアニメーションを実現できます。

1. GSAPの導入方法

CDNからの読み込み

最も簡単な導入方法は、CDNを使用してGSAPのライブラリを読み込むことです： 

<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.2/gsap.min.js"></script>

npm経由でのインストール

プロジェクトでnpmを使用している場合： 

npm install gsap

そして、JavaScriptファイルで以下のようにインポートします： 

import { gsap } from "gsap";

2. 基本的なアニメーションメソッド

gsapには、3つの基本的なアニメーションメソッドがあります。

gsap.to()

gsap.to()メソッドは要素を現在の状態から指定された状態へアニメーションさせます。

gsap.to() のデモ

gsap.to() のコード

// 現在の状態から指定した状態へアニメーション
gsap.to("#to-box", {
  x: 300,         // x方向に300px移動
  rotation: 360,  // 360度回転
  backgroundColor: "#e74c3c", // 色を変更
  duration: 2,    // 2秒かけて実行
  ease: "power2.out" // イージング
});

gsap.from()

gsap.from()は指定された初期状態から現在の状態へアニメーションさせます。

gsap.from() のデモ

gsap.from() のコード

// 指定した初期状態から現在の状態へアニメーション
gsap.from("#from-box", {
  x: -100,        // x方向に-100pxの位置から
  y: -50,         // y方向に-50pxの位置から
  opacity: 0,     // 透明な状態から
  scale: 0.5,     // 半分のサイズから
  rotation: -180, // -180度回転した状態から
  duration: 2,    // 2秒かけて実行
  ease: "power2.out" // イージング
});

gsap.fromTo()

gsap.fromTo()は指定された初期状態から指定された終了状態へアニメーションさせます。

gsap.fromTo() のデモ

gsap.fromTo() のコード

// 指定した初期状態から指定した終了状態へアニメーション
gsap.fromTo("#fromto-box",
  // 開始状態
  {
    x: -100,
    y: -50,
    opacity: 0,
    backgroundColor: "#9b59b6"
  },
  // 終了状態
  {
    x: 300,
    y: 0,
    opacity: 1,
    backgroundColor: "#2ecc71",
    duration: 2,
    ease: "power2.inOut"
  }
);

基本的な構文

gsap.to(targets, {
  // アニメーションさせたいプロパティと値
  x: 100,         // x方向に100pxの移動
  y: 50,          // y方向に50pxの移動
  rotation: 360,   // 360度の回転
  duration: 1,     // アニメーション時間（秒）
  ease: "power2.out", // イージング
  delay: 0.5,      // 遅延時間（秒）
  // その他のオプション...
});

パラメータの説明

  • targets: アニメーションさせたい要素（CSSセレクタ、DOM要素、配列など）
  • {}: 変数オブジェクト（アニメーションの設定を含む）
    • duration: アニメーションの時間（秒）
    • delay: アニメーション開始までの遅延時間（秒）
    • ease: イージング関数（速度変化のパターン）
    • その他のプロパティ: アニメーションさせたいCSS値

3. durationとdelay

GSAPアニメーションの時間的な要素を制御する重要なパラメータについて説明します。

duration（アニメーション時間）

durationパラメータは、アニメーションの再生にかかる時間を秒単位で指定します。

durationのデモ

durationのコード

// 速い（0.5秒）
gsap.to("#duration-box1", {
  x: 300,
  duration: 0.5,
  ease: "none"
});

// 普通（2秒）
gsap.to("#duration-box2", {
  x: 300,
  duration: 2,
  ease: "none"
});

// 遅い（4秒）
gsap.to("#duration-box3", {
  x: 300,
  duration: 4,
  ease: "none"
});

delay（遅延時間）

delayパラメータは、アニメーションが開始されるまでの遅延時間を秒単位で指定します。

delayのデモ

delayのコード

// 遅延なし
gsap.to("#delay-box1", {
  x: 300,
  duration: 1,
  delay: 0
});

// 1秒の遅延
gsap.to("#delay-box2", {
  x: 300,
  duration: 1,
  delay: 1
});

// 2秒の遅延
gsap.to("#delay-box3", {
  x: 300,
  duration: 1,
  delay: 2
});

その他の時間関連パラメータ

  • repeatDelay: リピート間の遅延時間（秒）
  • stagger: 複数要素のアニメーション開始時間をずらす（秒）

4. Transform系のプロパティ

GSAPではCSS Transform関連のプロパティを簡単に操作できます。これらは要素の位置、サイズ、回転などを制御するのに役立ちます。

位置の制御 (x, y, z)

要素の位置を移動させるには、x、y、zプロパティを使用します。

gsap.to(".element", {
  x: 100,  // 右に100px移動
  y: -50,  // 上に50px移動
  z: 20,   // z軸方向に20px移動（3D変換）
  duration: 1
});

回転 (rotation, rotationX, rotationY, rotationZ)

要素を回転させるには、次のプロパティを使用します：

gsap.to(".element", {
  rotation: 360,     // Z軸周りに360度回転（時計回り）
  rotationX: 45,     // X軸周りに45度回転
  rotationY: 30,     // Y軸周りに30度回転
  rotationZ: 15,     // Z軸周りに15度回転（rotationと同じ）
  duration: 2
});

拡大・縮小 (scale, scaleX, scaleY)

要素のサイズを変更するには、次のプロパティを使用します：

gsap.to(".element", {
  scale: 2,      // 全体を2倍に拡大
  scaleX: 1.5,   // 横方向のみ1.5倍に拡大
  scaleY: 0.8,   // 縦方向のみ0.8倍に縮小
  duration: 1.5
});

傾斜 (skew, skewX, skewY)

要素を傾けるには、次のプロパティを使用します：

gsap.to(".element", {
  skew: 20,     // X軸とY軸の両方を20度傾ける
  skewX: 15,    // X軸方向に15度傾ける
  skewY: 10,    // Y軸方向に10度傾ける
  duration: 1
});

変形の原点 (transformOrigin)

回転や拡大縮小の基準点を変更するには、transformOriginプロパティを使用します：

gsap.to(".element", {
  rotation: 360,
  scale: 1.5,
  transformOrigin: "left bottom", // 左下を基準に変形
  duration: 2
});

transformOriginには、「left」「center」「right」「top」「bottom」などの値や、パーセンテージ（「50% 50%」）、ピクセル（「100px 50px」）を指定できます。

5. イージング関数

イージング関数はアニメーションの速度変化のパターンを定義します。 GSAPでは多様なイージング関数が用意されています。

イージング関数のデモ

イージングのコード

// 直線 (一定速度)
gsap.to("#ease-box1", { x: 300, duration: 2, ease: "none" });

// 減速 (最初が速く、最後がゆっくり)
gsap.to("#ease-box2", { x: 300, duration: 2, ease: "power2.out" });

// 加速 (最初がゆっくり、最後が速く)
gsap.to("#ease-box3", { x: 300, duration: 2, ease: "power2.in" });

// 加減速 (最初と最後がゆっくり、中間が速い)
gsap.to("#ease-box4", { x: 300, duration: 2, ease: "power2.inOut" });

主なイージング関数

  • "none": 一定速度（線形）
  • "power1"、"power2"、"power3"、"power4": 指数関数的な変化
  • "back": 少し行き過ぎてから戻る
  • "elastic": 弾性のある動き
  • "bounce": バウンドする動き
  • "steps(n)": n段階のステップ状の変化

イージングの変化タイプ

  • "power2.in": 始めはゆっくり、徐々に加速
  • "power2.out": 始めは速く、徐々に減速
  • "power2.inOut": 始めと終わりがゆっくりで、中間が速い

6. repeatとyoyo

GSAPではアニメーションを繰り返したり、往復させたりする機能も提供しています。

repeat（繰り返し）

repeatパラメータは、アニメーションの繰り返し回数を指定します。-1を指定すると無限に繰り返します。

repeatのデモ

repeatのコード

// 1回繰り返し（合計2回再生）
gsap.to("#repeat-box1", {
  x: 300,
  duration: 1,
  repeat: 1
});

// 3回繰り返し（合計4回再生）
gsap.to("#repeat-box2", {
  x: 300,
  duration: 1,
  repeat: 3
});

// 無限繰り返し
gsap.to("#repeat-box3", {
  x: 300,
  duration: 1,
  repeat: -1
});

yoyo（往復）

yoyoパラメータをtrueに設定すると、繰り返し時にアニメーションが逆方向に再生されます。repeatと組み合わせて使用します。

yoyoのデモ

yoyoのコード

// 繰り返しのみ（yoyo無し）
gsap.to("#yoyo-box1", {
  x: 300,
  duration: 1,
  repeat: 3,
  yoyo: false
});

// yoyo（往復）あり
gsap.to("#yoyo-box2", {
  x: 300,
  duration: 1,
  repeat: 3,
  yoyo: true
});

// 無限yoyo
gsap.to("#yoyo-box3", {
  x: 300,
  duration: 1,
  repeat: -1,
  yoyo: true
});

repeatDelay（繰り返し間の遅延）

repeatDelayパラメータを使用すると、繰り返し間に遅延を設けることができます。

gsap.to(".element", {
  x: 100,
  duration: 1,
  repeat: 2,
  repeatDelay: 0.5, // 繰り返しの間に0.5秒の遅延
  yoyo: true
});

GSAPの基本構文まとめ

  1. gsap.to(): 現在状態から指定した状態へアニメーション
  2. gsap.from(): 指定した状態から現在状態へアニメーション
  3. gsap.fromTo(): 指定した開始状態から終了状態へアニメーション
  4. durationとdelay: アニメーションの時間と遅延を制御
  5. transform系のプロパティ: 位置、回転、拡大縮小などを制御
  6. イージング関数: アニメーションの速度変化パターンを制御
  7. repeatとyoyo: アニメーションの繰り返しと往復を制御

これらの基本を組み合わせることで、様々な動きを表現できます。 実際に手を動かして試してみることが上達の近道です。
